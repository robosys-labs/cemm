# CEMM v3.5.1 Phase 4–5 Implementation

Target baseline: `3120fbdfc2c786a10adcdfe5bd5248658c843069`.

## Phase 4 — complete stabilization substrate

### 4A. Compile semantic eligibility once per AuthorityGeneration

Implemented `CompiledSemanticCapabilityRegistry`.

It compiles exact:
- record pin;
- lifecycle/use profile;
- record-family use compatibility;
- dependency lineage;
- promotion/use-grant lineage;
- authority generation/fingerprint.

Overlay records carrying learning-promotion dependencies require an exact singular
candidate/promotion/use-grant lineage. Reviewed non-learning overlays are not forced
through the learning promotion protocol merely because they live in the overlay. All
executable authority must still be the singular effective active revision for its
ref/kind. Ambiguous active branches and invalidated revisions are fail-closed.
Semantic eligibility is not an effect grant.

`LanguageUseAuthority` now delegates to this compiled registry instead of independently
rescanning active revisions, dependency edges and promotion decisions on each use.

### 4B. Narrow effect authorization

Implemented `EffectAuthorizationBoundary` + `AuthorizedEffectStore` for:
- pre-effect exact-patch durable persistence (world/discourse/audit/effect-journal changes);
- external operation;
- protected disclosure;
- external emission.

It is intentionally separate from semantic eligibility. Runtime services receive a
read-only store view plus `AuthorizedEffectStore`; raw `apply_patch()` is unavailable.
Every durable GraphPatch requires a pre-effect receipt bound to exact patch ref/fingerprint,
CAS store revision, target set, authority generation and permission scope. The orchestrator
verifies a complete receipt chain for every committed store revision; Stage 16/20 additionally require operation/disclosure/emission receipts.
Stage 22 may consolidate and schedule promotion, but executable authority publication is
a post-pass maintenance boundary because the current pass holds an immutable
AuthorityGeneration lease.

### 4C. Cycle-local transient record view

Implemented `WorkspaceRecordIndex` + `CycleArtifactStoreView`.

Exact-pin planners may consume transient Stage-14/15/18/19 artifacts without forcing
those stages to persist. Transient executable semantic authority is explicitly
forbidden.

### 4D. Proof-carrying realization

Implemented representation-neutral:
- `RealizationTransformStep`;
- `RealizationProof`;
- `RealizationProofBuilder`;
- `RealizationProofVerifier`;
- `SemanticPreservationAssessment`.

Proofs carry exact semantic input fingerprint, authority generation, transform pins,
coverage, qualification fingerprint, permission/audience and surface hash.

The proof module has no UOL import and is usable by future Response CSIR.

### 4E. Selective independent round-trip

Implemented `SelectiveRoundTripPolicy`.

Cheap proof verification is mandatory for every selected emission candidate and is
enforced by the Stage-20 kernel wrapper before an emitter service runs. Independent
re-analysis is required for novelty/high-risk/audit/release-competence/unreviewed-
transform/channel-transform cases. A failed cheap proof cannot be rescued by full
round-trip. Stage 20 separates read-only emission authorization from actual `emit()`;
protected-disclosure and emission receipts are issued before the external effect, and
any observed emission must identify the proof-verified selected candidate.

### 4F. Event boundary correctness

The new canonical runtime drains targeted maintenance events only after the semantic
pass AuthorityGeneration lease is released. It does not run general learning scans
merely because a request happened.

### 4G. M0 and legacy/performance gates

Added:
- `tools/check_v351_legacy_boundaries.py`;
- `tools/check_v351_m0_gate.py`;
- Phase-4 capability/effect/proof tests.

M0 remains unproven until those commands and the complete regression suite pass.
No release hashes are regenerated by this phase.

## Phase 5 — exact v3.5.1 Stage ABI

### 5A. Exact Stage enum

`CoreStage` now matches `CORE_LOOP.md` exactly, including:

```text
5 COMPILE_CANDIDATES_TO_CSIR
6 RUN_RECURRENT_MEANING_DYNAMICS
7 STABILIZE_SEMANTIC_ATTRACTORS
18 CONSTRUCT_RESPONSE_CSIR
```

There are no old UOL stage aliases in the canonical ABI.

### 5B. Machine-readable StageContract

Every stage declares:
- required inputs;
- produced outputs;
- required outputs on successful execution;
- persistence class;
- mutable generations allowed;
- effects allowed;
- frontier classes;
- budgets;
- proof requirements.

`runtime_graph.py` is derived from these contracts, removing stale hand-maintained
mutation booleans as the source of truth.

### 5C. New StageCapability and CognitiveCycleState

`StageCapability` pins:
- `cycle_ref`;
- `pass_ref`;
- exact stage/predecessor;
- nonce;
- AuthorityGeneration/fingerprint;
- exact ReadGeneration.

No `CycleState` alias is exported. Stale v3.5 callers must migrate rather than silently
receiving changed semantics.

### 5D. Persistence/effect enforcement

The orchestrator compares generation changes before/after each stage and rejects any
change outside the StageContract matrix.

Thus a Stage-14/15/18/19 accidental world/authority write is a runtime contract
violation even if stale code attempts it.

### 5E. Concrete Stage 0–22 adapters

All adapters are explicit classes with `v351.stage.*` identities and exact handler
ownership. Capability ownership validates cycle + pass + predecessor.

### 5F. No UOL fallback at Stages 5–7

The canonical runtime imports no `.uol`, `.composition`, `v347`, old runtime hardening
or old MeaningComposer.

Stages 5–7 require explicit services:
- `csir_compiler` -> `CSIRCandidateSet`;
- `recurrent_semantic_solver` -> `ActivationGraph`, `ActivationTrace`;
- `semantic_attractor_stabilizer` -> `SemanticAttractorSet`.

If absent, the stage emits a typed runtime-capability frontier. It never calls the
legacy factor-graph/UOL pipeline.

This is intentionally not pretending Phase 6 is already implemented.

### 5G. Bounded re-entry/generation rules

Only Stage 17 may request semantic re-entry. Re-entry receives a fresh pass ref and
explicit carry whitelist. Read-generation restarts remain bounded. The pass pins one
pre-commit cognitive generation; concurrent drift restarts through Stage 13 and fails
closed to replay after commit instead of silently floating to a new world/discourse view.

### 5H. Legacy runtime and test regression boundaries

- `cemm.v347` no longer re-exports `Runtime` or wildcard model semantics, and the
  executable `cemm/v347/runtime.py` entrypoint is removed from the active tree.
- The old UOL-oriented `runtime.py` is overwritten, not retained as executable fallback.
- Legacy UOL/composition modules may remain only for explicit future migration/shadow
  compiler work.
- Architecture lint rejects canonical direct imports of v347/UOL/composition/old
  hardening. v3.5 UOL record codecs/projectors that still exist before Phase 8 are
  migration substrate only and cannot be selected as runtime services or fallback brain.
- The old `runtime_services.py` semantic composition root is tombstoned; signed v351
  services must declare `RUNTIME_ABI = "v351"` and matching `SERVICE_KIND`.
- Tests that still assert old Stage ABI or import v347/legacy runtime services are
  reported, never silently skipped.

### 5I. Acceptance/doc contract reset

The UOL-era `ACCEPTANCE_CONTRACT.md` is archived and replaced with a v3.5.1 CSIR
contract. Superseded root plans are archived so `IMPLEMENTATION_PLAN.md` remains the
single roadmap.

## Important capability state after Phase 5

Phase 5 establishes the real ABI and removes automatic fallback. It does **not** invent
the Phase-6 CSIR kernel or recurrent semantic implementation.

Therefore, until Phase 6–8 install exact CSIR/compiler/recurrent services, a canonical
public cycle may defer at Stage 5 with `runtime_capability:csir_compiler`.

That is expected and architecturally safer than keeping the old UOL brain authoritative.
