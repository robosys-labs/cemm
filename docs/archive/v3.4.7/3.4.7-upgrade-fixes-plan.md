# CEMM v3.4.7 Foundational Breakthrough Implementation Plan

**Status:** canonical replacement implementation plan  
**Architecture:** `architecture.md`  
**Runtime contract:** `coreloop.md`  
**Goal:** replace sentence-recognition authority with a working multilingual, grounded, teachable UOL semantic kernel  
**Non-goal:** cosmetically renaming v3.4.6 modules while retaining the existing construction-authorized behavior.

---

## 1. Executive decision

v3.4.7 is a semantic-authority migration, not an incremental language-pack release.

The current runtime has valuable foundations—typed schemas, lifecycle activation, provenance, bounded inference, learning transactions, and proof-oriented realization—but the primary understanding path remains dependent on declarative ordered construction matches. The “semantic forest” enriches output produced by the prior composer rather than replacing that authority. Context, grounding, goals, operations, and responses then operate on candidates already constrained by that sentence-pattern path.

The upgrade must therefore replace the center of gravity:

```text
FROM
surface/construction match
  → predication
  → ranking and response patches

TO
multimodal evidence lattice
  → Referent candidates
  → schema activation and local ports
  → bounded UOL hypothesis assembly
  → compatible MeaningBundle
  → knowledge/learning/goals/response
```

The release is complete only when old composer authority is physically removed from the canonical runtime and the baby-CEMM proof suite passes without transcript-specific constructions.

## 2. Second-pass audit: weaknesses in the first v3.4.7 draft

The first draft was directionally correct but still left openings for a shallow makeover.

### 2.1 Split semantic-object terminology

It retained separate top-level terms for referents, values, propositions, states, quantities, units, and time. That encourages special stores, filler branches, and resolver paths.

**Correction:** adopt `Referent` as the single identity-bearing semantic object family. Use `ReferentKind` and specialized canonical payloads. All predicate ports bind referents.

### 2.2 UOL was too response-oriented

The draft described a UOL response plan but did not make UOL the canonical temporary workbench for understanding, ambiguity, and multimodal composition.

**Correction:** introduce cycle-local `UOLGraph`, `MeaningHypothesis`, `MeaningBundle`, and durable-change `GraphPatch`. The outbound `UOLResponsePlan` is a selected semantic projection from the same substrate.

### 2.3 Operational ports still depended too much on global roles

Deriving operational ports from globally reusable role schemas permits generic role drift and hidden actor/object assumptions.

**Correction:** ports are owned locally by `PredicateSchema` and `OperationSchema`. Optional role families support cross-schema alignment, but exact semantics and constraints belong to `named.holder`, `located_at.place`, `move.destination`, and similar local ports.

### 2.4 The meaning assembler algorithm was underspecified

“Schema-driven composition” can still become a wrapper around existing candidates unless activation sources, port candidate pools, hard constraints, beam limits, score factors, and bundle compatibility are explicit.

**Correction:** implement a bounded joint referent/port solver and compatible-set selector with named traces and no fallback to canonical composer output.

### 2.5 Foundation seeding was not an implementation gate

A new runtime could be wired with an ontology too thin to understand age, quantities, time, roles, state, event participation, or communication.

**Correction:** make the operationally complete foundation package a blocking early phase with semantic competence tests.

### 2.6 “Working baby CEMM” was not measurable enough

Passing the old demo would still allow overfitting.

**Correction:** require cross-language, paraphrase, conjunction, non-entity anaphora, recursive teaching, safe rule classification, operation binding, multimodal reference, and emission-blocking tests.

### 2.7 Learning and causal recursion needed stricter logical discipline

The first plan supported recursive rules but did not sufficiently use relationship examples to prove that strict, default, causal, statistical, and sensitive relations remain separate.

**Correction:** add explicit rule-class tests and prevent sensitive/default materialization into actual facts.

### 2.8 Knowledge still risked duplicating UOL

Retaining predicate/role blobs in a fact store alongside proposition objects creates parallel semantic identities.

**Correction:** target `KnowledgeRecord(proposition_ref, epistemic metadata)` and migrate old `SemanticFact` records through a compatibility layer.

## 3. Non-makeup release gates

The following are hard gates. No phase or release may be called complete while any is false.

1. The canonical runtime does not call `CanonicalSemanticComposer` or `SemanticForestComposer` to create authoritative predications.
2. The construction matcher is an evidence provider only.
3. Kernel code contains no transcript phrase routing or language-specific word tests.
4. `Referent` is the only semantic port-filler family.
5. Predicate and operation ports are local schema definitions.
6. Selected meaning is a compatible bundle, not one top proposition.
7. Generic gaps do not default to learnable.
8. All durable semantic changes use GraphPatches.
9. Defaults, associations, and sensitive inferences cannot silently materialize actual facts.
10. Response candidates are generated semantically before ranking.
11. The renderer cannot construct new meaning.
12. At least two typologically different languages pass the same UOL tests.
13. The old demo and neighboring paraphrases pass without adding exact full-sentence constructions.
14. Status claims distinguish specified, implemented, wired, authoritative, and verified.

## 16. Phase 10 — Generalized grounded learning and GraphPatch discipline

### Objective

Make learning a first-class semantic loop that can acquire new language and concepts safely.

### Workstream 10A: Contribution classifier

Classify lexical aliases, identities, kinds, relationships, predicates/ports, states, units, event/place patterns, operations, strict/causal/enabling/default/statistical rules, exceptions, and realizations.

### Workstream 10B: Grounding frontier

Create exact dependencies and recursive questions. Prioritize the question with highest expected frontier reduction and lowest risk.

### Workstream 10C: General compiler

Replace definition-only special cases with compilers that produce typed GraphPatches for every contribution family.

### Workstream 10D: Multilingual lexical learning

A new surface in any supported language can link to an existing semantic schema. If it denotes a new concept, recurse through kind/relationship/predicate anchors before activation.

### Workstream 10E: Scope and promotion

Default new theories to session/private scope. Require independent evidence or review for wider promotion.

### Workstream 10F: Competence and replay

Generate positive, negative, contrastive, and paraphrase tests that are not merely copied from teaching examples. Replay affected turns after activation.

### Workstream 10G: Learning dialogue

Ensure ordinary new assertions are not consumed as answers to unrelated learning probes. Support meta-questions, corrections, cancellation, and resume.

### Required learning demonstrations

1. Teach an unknown word as an alias to a known referent kind.
2. Teach a new entity/role concept grounded in existing foundations.
3. Teach a new predicate with typed ports.
4. Teach a rule and classify its strength/class.
5. Teach a second-language lexicalization.

### Exit criteria

- every learned executable item has grounding anchors and provenance;
- no raw text field becomes an executable semantic definition;
- learning can span several turns and resume correctly;

## 23. Working baby CEMM release proof

The release candidate must demonstrate this end-to-end sequence in one stable session, plus equivalent/paraphrased and second-language cases.

### 23.1 Self and seeded knowledge

```text
User: What is your name?
CEMM: My name is CEMM.

User: What does it mean?
CEMM: It means Contextual Event Memory Model.
```

Proof requirements:

- self and name are referents;
- `it` resolves to the salient name/acronym proposition or name referent;
- answer comes from seeded knowledge;
- no learning transaction opens.

### 23.2 User identity and state

```text
User: My name is Chibueze Opata and I am 34 years old.
CEMM: [grounded acknowledgement]

User: What is my name and how old am I?
CEMM: [answers both]
```

Proof requirements:

- two coordinated propositions;
- name, quantity, unit, time-indexed age;
- atomic knowledge patch;
- multi-answer response bundle.

### 23.3 Reference and correction

```text
User: You keep saying that.
```

`that` must rank recent proposition referents. If ambiguous, CEMM asks which statement.

A later name/age correction must supersede the exact prior proposition/state without deleting historical evidence.

### 23.4 Grounded teaching

Teach a new term in either supported language. CEMM must ask only necessary frontier questions, anchor the concept, store it in session scope, and use it compositionally in a new sentence.

### 23.5 Rule learning

Teach a role/default/causal rule. CEMM must identify its class or ask. It must derive only allowed consequences and qualify defaults.

### 23.6 Operation

Issue a safe operation request with two referent ports. CEMM must resolve/bind, clarify ambiguity if needed, authorize, execute/simulate, reconcile, and report the outcome.

### 23.7 Unknown and unsupported input

When partial meaning is available, respond to that meaning or ask about the exact unresolved part. When no authorized semantic plan can be built, do not fabricate a fluent answer.

## 26. Risk register and controls

### Combinatorial explosion

**Control:** candidate caps, hard constraint propagation, indexed activation, clause-local beams, ambiguity-family retention, wall-clock budgets, traceable pruning.

### Statistical analyzer becomes authority

**Control:** analyzers emit evidence contracts only; semantic kernel revalidates through schemas and ports; architecture import lint.

### New UOL becomes a giant persistent sentence graph

**Control:** UOLGraph is cycle-local; durable writes require compressed GraphPatches; knowledge points to admitted propositions only.

### Referent unification becomes type erasure

**Control:** strict ReferentKind/type schemas and specialized canonical payloads; ports retain accepted kinds and types.

### Foundation becomes domain dump

**Control:** structural admissibility, small operational seed, domain concepts learned in scoped packages, no examples auto-promoted.

### Plausibility creates stereotypes or false facts

**Control:** plausibility ranks only compatible candidates; rule class/consequence status; explicit assumptions; sensitive inference policy.

### Learning pollutes global schemas

**Control:** session/private default, grounding frontier, independent competence, CAS activation, review/promotion pipeline, rollback.

### Response fluency hides missing understanding

**Control:** semantic response goals, UOL plan, full coverage proof, no renderer content decisions, no-output candidate.

### Rewrite stalls

**Control:** vertical slices below, one canonical model early, migration adapters, measurable phase exits, old path shadow-only then removal.
