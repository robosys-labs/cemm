# CEMM v3.5 — Learning-First Contextual Event Memory Model

CEMM is a grounded semantic kernel designed to learn reusable meaning from multimodal evidence and express only what it can justify.

## Core idea

CEMM models every identifiable thing as a `Referent`. Referent types provide inherited **facet entitlements** describing which properties, states, relations, roles, functions, affordances, and capabilities can apply.

Meaning is composed in UOL from typed schemas and operators. Events change referent state through generic transition contracts. Claims remain source-attributed until epistemically assessed. Impact and importance guide response goals without becoming facts.

```text
evidence
→ referent/schema candidates
→ type and facet projection
→ UOL composition
→ selected meaning
→ claims/knowledge/learning
→ event transitions and capability effects
→ impact/importance
→ goals/actions
→ response UOL
→ multilingual grammar
→ verified output
```

## Key distinctions

CEMM keeps these separate:

```text
claim ≠ fact
property ≠ state
affordance ≠ capability
capability ≠ permission
function ≠ current ability
truth negation ≠ harmful valence
loss ≠ scalar decrease
importance ≠ mention frequency
event result ≠ state effect
```

## Learning first

A learned concept can add:

- semantic type and inheritance;
- facet entitlements;
- properties and state dimensions;
- actions/events and participant ports;
- transition effects;
- affordances and capability dependencies;
- rules, defaults, examples, and exceptions;
- lexical senses and multilingual realization.

Learned data enters the same ordinary indexes as seed data and survives restart.

## Release target

v3.5 is successful when new referent types and events can be learned, can alter state through generic semantics, can affect capabilities and significance, can guide response goals, and can be realized in multiple languages without new kernel branches or full-sentence templates.
