# v3.4.3 architecture delta

## 1. Foundations are executable contracts

The internal label `requires_for_goal` is not treated as a self-explanatory
English primitive. Its contract defines roles, identity, query behavior,
contradiction behavior, context/time behavior, permitted operations, property
cases, and implementation binding.

Ordinary boot schemas such as `information_object`, `goal`, `requirement`,
`agent`, and `software_system` remain ordinary schema revisions. Each has a
constitutive pattern, identity criteria, operational ports, dependencies, and
independent competence cases.

## 2. Goal requirements are real cognitive states

A current requirement exists only when:

```text
active_or_blocked(goal)
∧ held_by(goal, requirer)
∧ depends_on(goal, specification)
∧ not_satisfied(specification, context, valid_time)
```

Therefore, a plan containing `requires_for_goal(self, ...)` cannot be generated
or realized from a lexical gap alone.

## 3. Semantic emission closure

The response path becomes:

```text
semantic message plan
→ foundation contract check
→ role/type/provenance check
→ self-claim authorization
→ construction selection
→ lexical/morpheme competence check
→ round-trip evidence check
→ realization with full span coverage
→ dispatch
→ common-ground commit
```

No dispatch occurs when any required proof component is missing.

## 4. Language packs do not own truth

Language packs contribute reversible lexical, grammatical, construction, and
realization evidence. They do not create goals, requirements, facts,
capabilities, learning outcomes, or causal warrant.

## 5. Recursive learning changes the correct artifact family

Learning may compile:

```text
entity kind
role
predicate
state dimension
event kind
operation
rule
lexicalization
construction
realization
```

It does not default every teaching turn to a `LexemeSenseSchema`.

Activation requires at least one operational attachment to an executable
foundation or valid active schema.

## 6. Inference distinguishes semantic operators

```text
strict entailment
defeasible default
probabilistic association
causal claim with warrant
```

Existential consequences produce constraints rather than invented people,
places, or events. Sensitive probabilistic associations are disabled unless a
policy explicitly permits them.

## 7. Core loop integration

```text
ORIENT
UNDERSTAND
KNOW
  derive truth, live goals, requirements, self assessments
DECIDE
ACT AND RECONCILE
  bounded inference/learning/operations
CRITICAL COMMIT
COMMUNICATE
  emission closure
OUTPUT COMMIT
```
