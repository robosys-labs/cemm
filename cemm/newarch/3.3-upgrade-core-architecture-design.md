# CEMM 3.3 Upgrade: Core Architecture Design

**Status:** Proposed canonical 3.3 architecture — learning-integrated revision 2  
**Baseline reviewed:** `robosys-labs/cemm@c6675ee5e822c4af6253e1bce2e76975fed0f9c9`  
**Scope:** Multilingual perception, recursive semantic grounding, online vocabulary/schema learning, interpretation, entity grounding, state transmutation, causality, obligations, contracts, execution, memory revision, and response formation.  
**Non-goal:** Transcript-specific phrase routing, raw-text memorization, or using an LLM as an untraceable semantic authority.

---

## 1. Executive correction

CEMM's highest leverage is not that it can answer from a fixed ontology. It is that it can **acquire new language and new operational meaning while remaining structurally grounded**.

The 3.3 architecture therefore has two equal online spines and one slower consolidation loop:

```text
EXECUTION SPINE
surface signal
-> normalized multilingual evidence
-> meaning groups and interpretation branches
-> grounded UOL propositions
-> scoped operator/state/relation activation
-> state transmutations and causal effects
-> obligation graph and execution contracts
-> response/action/state outcomes

ACQUISITION SPINE
unknown or uncertain evidence
-> typed semantic gap
-> learning episode
-> minimum-information question
-> user/source/multimodal evidence
-> provisional binding or schema
-> immediate scoped reuse
-> reinforcement, contradiction, revision, or retirement

CONSOLIDATION LOOP
provisional learning + usage outcomes + independent evidence
-> evidence aggregation
-> sense/schema separation
-> confidence/stability update
-> graph-patch validation
-> durable promotion, supersession, quarantine, or rejection
```

Learning is not a final pipeline phase. It participates at normalization, perception, grounding, state, operator activation, query, response, and post-turn revision.

## 2. Core architectural laws

### 2.1 Surface evidence never directly becomes authority

A token, phrase, named entity, morphology guess, alias match, article, preposition, or construction match is evidence for one or more interpretations. It is not itself an action, fact, state mutation, query, write, or safety event.

```text
surface form -> lexical/grammatical candidates -> selected scoped meaning
```

### 2.2 Unknown meaning is first-class

Unknown or low-confidence material is represented as a `SemanticGap`, not forced into `social_act`, `world_fact_claim`, `concept_definition_query`, or an invented entity/action.

A gap records exactly what is missing:

```text
lexeme sense
entity kind or identity
predicate/operator identity
argument/port filler
relation orientation
state family/dimension/value
modality/polarity/evidentiality
temporal or geospatial anchor
grammatical function
construction mapping
causal effect
```

### 2.3 Learning targets fundamental substrates

CEMM recursively asks until it can bind the unknown to existing substrates or propose a bounded schema extension:

```text
entity / entity kind
concept
relation
operator + typed ports
state family + dimension + value/polarity/scale
time / place / geospatial relation
source / permission / modality
grammatical feature or construction
causal/affordance effect
realization pattern
```

The system does not need a dictionary-style gloss if it can acquire an operational graph mapping.

### 2.4 Learning and truth are separate

Learning that a surface form maps to a meaning is distinct from accepting a world claim expressed with it.

```text
"zorba" means "move quickly"      -> lexical/operator binding candidate
"Zorba happened yesterday"        -> separate event/world assertion
```

A reliable lexical mapping does not make every proposition using it true.

### 2.5 Authority progression is monotonic and traceable

Every artifact has an authority state:

```text
observed -> candidate -> selected -> scoped -> validated
-> authorized -> executed -> committed
```

Alternative terminals are:

```text
unresolved, suppressed, quarantined, contradicted, rejected, retired
```

No component may silently promote authority.

### 2.6 Immediate use is scoped, not globally trusted

A newly taught meaning may become usable immediately through a **session provisional overlay** once minimally grounded. Global/durable promotion requires stronger evidence.

```text
session_provisional -> user_scoped -> domain_scoped -> language_scoped/global
```

### 2.7 Contradiction creates revision, not destructive overwrite

A new observation may:

```text
reinforce an existing sense
weaken it
split a new sense
add a context restriction
supersede a time-bounded meaning
quarantine a conflict
```

The evidence ledger remains append-only; materialized confidence is derived.

### 2.8 Grammar is operational meaning

Articles, determiners, prepositions, case markers, auxiliaries, inflections, particles, conjunctions, and word order are not filler to discard. They can encode:

```text
definiteness / specificity / quantity
time / aspect / completion
source / goal / instrument / possession
geospatial topology
modality / negation / evidentiality
discourse relation / information structure
```

They are represented as language-specific grammatical bindings to language-neutral semantic features and graph rewrites.

### 2.9 Online learning cannot grant execution permission

A user may teach what a new word means. That learning cannot create tool authority, bypass safety, change foundational UOL primitive kinds, or install a globally trusted high-risk operator without independent validation.

### 2.10 All durable learning is graph-patch mediated

Perception, clarification, and use may create candidates and session overlays. Durable mutation occurs only through provenance-bound `GraphPatch` operations that pass validation, authorization, conflict checks, and scope policy.

## 3. Canonical 3.3 runtime

```text
0. Restore ContextKernel, SessionStore, active LearningEpisode, provisional overlays
1. Normalize signal by language/script/morphology; preserve original evidence
2. Segment meaning groups; emit lexical, grammatical, NER, and construction candidates
3. Build interpretation lattice and UOL candidate subgraphs
4. Ground entities/references and resolve discourse, time, place, source, permission
5. Detect SemanticGap objects and determine whether each gap blocks execution
6. Activate only scoped predicates/operators with valid typed ports
7. Compile state occupancy/delta/transmutation candidates
8. Compile operational meaning and causal effects per selected branch
9. Build compatible obligation graph
10. If a blocking gap exists, add a learning/clarification obligation
11. Compile query/write/reaction/safety/state/learning contracts once
12. Execute contracts transactionally and record ExecutionLedger
13. Form response from selected evidence and execution results
14. Apply output/session state and provisional learning overlays
15. Observe success, correction, reuse, contradiction, and downstream prediction
16. Emit learning/revision GraphPatch candidates
17. Persist session; consolidate durable candidates asynchronously or within budget
```

Recursive clarification is normally **turn-recursive**, not an unbounded function call. Each turn asks one highest-value question, persists the episode, and consumes the next answer as evidence under that episode.

## 4. Dual-loop runtime contracts

### 4.1 `SemanticGap`

```python
@dataclass(frozen=True, slots=True)
class SemanticGap:
    gap_id: str
    branch_id: str
    group_id: str
    span_ref: "SemanticRef"
    language_tag: str
    gap_kind: str
    # lexeme_sense, grammatical_function, entity_kind, entity_identity,
    # operator_identity, required_port, relation, state_coordinate,
    # temporal_anchor, geospatial_anchor, modality, causal_effect, construction

    blocking_artifact_ids: tuple[str, ...]
    required_fields: tuple[str, ...]
    resolved_fields: Mapping[str, Any]
    candidate_hypothesis_ids: tuple[str, ...]
    entropy: float
    confidence: float
    provenance: "ProvenanceEnvelope"
```

A gap is blocking only when unresolved meaning is required for a selected obligation, safety decision, query target, write target, or state transition. Non-blocking gaps remain traceable while the rest of the turn proceeds.

### 4.2 `LearningEpisode`

```python
@dataclass(slots=True)
class LearningEpisode:
    episode_id: str
    context_id: str
    target_scope: str
    target_gap_ids: list[str]
    hypothesis_ids: list[str]
    evidence_event_ids: list[str]
    asked_question_ids: list[str]
    unresolved_fields: set[str]
    provisional_binding_ids: list[str]
    recursion_depth: int
    status: str
    # active, minimally_grounded, provisionally_active, awaiting_evidence,
    # consolidated, abandoned, quarantined, rejected
    expires_at: float | None
```

Only one episode owns a gap. One episode may cover several dependent gaps when one answer can resolve them together.

### 4.3 `LearningHypothesis`

```python
@dataclass(frozen=True, slots=True)
class LearningHypothesis:
    hypothesis_id: str
    target_kind: str
    proposed_artifact: Any
    satisfied_fields: tuple[str, ...]
    missing_fields: tuple[str, ...]
    applicability_context: "ContextSignature"
    score: "KnowledgeStrength"
    provenance: "ProvenanceEnvelope"
```

### 4.4 `LearningEvidenceEvent`

```python
@dataclass(frozen=True, slots=True)
class LearningEvidenceEvent:
    event_id: str
    target_hypothesis_id: str
    evidence_kind: str
    # explicit_definition, example, counterexample, translation,
    # multimodal_alignment, independent_recurrence, successful_use,
    # user_confirmation, user_correction, prediction_failure

    stance: str                 # support, contradict, restrict, supersede
    weight: float
    source_ref: "SemanticRef"
    source_trust: float
    independence_key: str
    language_tag: str
    context_signature: "ContextSignature"
    observed_at: float
    provenance: "ProvenanceEnvelope"
```

Repeated evidence from the same source/session/context is deduplicated or dependency-discounted.

### 4.5 `KnowledgeStrength`

A single mutable confidence number is insufficient.

```python
@dataclass(frozen=True, slots=True)
class KnowledgeStrength:
    semantic_confidence: float
    source_trust: float
    support_mass: float
    contradiction_mass: float
    source_diversity: float
    context_coverage: float
    language_coverage: float
    successful_use_rate: float
    repair_failure_rate: float
    stability: float
    freshness: float
```

`semantic_confidence` is derived from the ledger. Promotion policy also considers coverage, trust, independence, stability, and risk.

### 4.6 `LearningContract`

```python
@dataclass(frozen=True, slots=True)
class LearningContract:
    contract_id: str
    episode_id: str
    target_hypothesis_ids: tuple[str, ...]
    permitted_operations: tuple[str, ...]
    activation_scope: str
    promotion_ceiling: str
    required_evidence: tuple[str, ...]
    prohibited_authorities: tuple[str, ...]
    provenance: "ProvenanceEnvelope"
```

This is the sole authority for creating provisional bindings, evidence events, revision operations, and durable learning patches.

## 5. Learnable artifact hierarchy

CEMM learns different artifact classes under different minimum-grounding and promotion rules.

### 5.1 Surface and morphology

```text
SurfaceFormBinding
LemmaBinding
MorphologicalRuleCandidate
Script/orthographic normalization candidate
```

These bind observed forms to sense candidates without claiming operational meaning by themselves.

### 5.2 Lexeme and sense

```python
LexemeSenseBinding(
    language_tag,
    dialect/domain,
    normalized_form,
    lemma,
    grammatical_category,
    semantic_target_ref,
    sense_context,
    strength,
    scope,
)
```

One surface may map to several senses. Context selects a branch; no destructive single-definition slot is used.

### 5.3 Grammar and constructions

```text
GrammarOperatorBinding
ConstructionSchema
ArgumentRealizationSchema
InflectionRule
WordOrderConstraint
DiscourseConstruction
```

A grammatical binding maps a surface/morpheme/position to semantic features or graph operations, for example:

```text
determiner -> definiteness/specificity/quantity
preposition/case marker -> source/goal/location/instrument/domain/possession
tense/aspect marker -> temporal validity/completion
negator -> polarity scope
modal -> desired/possible/obligatory/reported
conjunction -> discourse/causal/conditional relation
```

### 5.4 Concepts, entities, relations, states, and operators

```text
ConceptCandidate
EntityKindCandidate
EntityIdentityBinding
RelationSchemaCandidate
StateLexemeBinding
StateDimensionCandidate
OperatorSchemaCandidate
Affordance/CausalRuleCandidate
```

A new action/operator is not minimally grounded until its required typed ports and operational effect are sufficiently known.

### 5.5 Realization knowledge

```text
SemanticUnitLexicalization
ConstructionRealizationSchema
Agreement/ordering rule
Register/style variant
```

Realization learning maps already-authorized semantic units to a language. It never decides turn meaning.

## 6. Minimum sufficient grounding

The learning loop stops asking when the target artifact is usable at the requested scope.

| Artifact | Minimum session-provisional grounding | Additional durable/global requirements |
|---|---|---|
| Alias to known concept/entity | language, form, target sense | independent recurrence or trusted definition; context coverage |
| New noun/concept | entity/concept kind plus differentiating relation/property | stable identity/taxonomy; contradiction check |
| New proper name | entity identity/type and referent scope | user/domain scope; collision resolution |
| New verb/action | operator family, actor/target ports, modality behavior, at least one effect/relation delta | typed preconditions, examples/counterexamples, risk/permission review |
| New adjective/state term | holder kind, state family, dimension, value/polarity/scale | repeated contextual alignment; schema compatibility |
| New relation/preposition | source/target roles, relation orientation, context | contrastive examples; inverse/composition policy |
| Article/determiner | grammatical function and scoped semantic feature | distribution across constructions and noun classes |
| Construction | slot pattern and UOL graph rewrite | diverse lexical fillers; low collision rate |
| Causal rule | antecedent, effect, scope, confidence | independent evidence; temporal/contradiction validation |
| Realization form | semantic unit and language output form | grammaticality/use success across contexts |

The system may use an artifact below durable threshold only within the contract's scope and with uncertainty preserved.

## 7. Recursive grounding protocol

### 7.1 Gap localization

The system first asks: **where is the missing meaning load?**

```text
unknown span is predicate head   -> operator/relation gap
unknown span fills an argument   -> entity/concept/kind gap
unknown modifier                 -> state/quality/quantity gap
unknown closed-class item        -> grammatical-function gap
known words but incoherent graph -> construction/scope gap
resolved proposition but unknown effect -> causal/operator-schema gap
```

NER, morphology, word position, dependency candidates, surrounding known operators, port expectations, state schemas, and discourse context produce this localization. They nominate a gap kind; they do not finalize it.

### 7.2 Question planning

`LearningQuestionPlanner` chooses one semantic question act maximizing expected uncertainty reduction per interaction cost:

```text
utility = expected_entropy_reduction
          * downstream_blocking_weight
          * reuse_value
          * safety_weight
          / question_cost
```

Question acts include:

```text
ASK_SEMANTIC_KIND
ASK_REFERENT_IDENTITY
ASK_OPERATOR_ACTOR
ASK_OPERATOR_TARGET
ASK_OPERATOR_EFFECT
ASK_RELATION_ORIENTATION
ASK_STATE_DIMENSION
ASK_STATE_VALUE_OR_POLARITY
ASK_TEMPORAL_SCOPE
ASK_GEOSPATIAL_RELATION
ASK_GRAMMATICAL_FUNCTION
ASK_EXAMPLE
ASK_COUNTEREXAMPLE
ASK_TRANSLATION_EQUIVALENT
ASK_CONFIRM_HYPOTHESIS
```

NLG realizes these acts in the conversation language. The planner never contains English prompt strings.

### 7.3 Constrained answer interpretation

The next user turn is not parsed as an unrelated generic utterance. The active episode supplies an expectation frame:

```text
expected answer kind
open semantic fields
candidate hypotheses
referent and topic bindings
question provenance
```

Normal perception still runs, but episode constraints improve ranking rather than replacing evidence.

### 7.4 Provisional activation

When minimum grounding is met:

1. create a provisional artifact candidate;
2. validate internal consistency and schema compatibility;
3. activate it in the session overlay;
4. re-run only affected interpretation branches;
5. answer or continue the original obligation when possible;
6. retain the episode for later reinforcement if durable threshold is unmet.

### 7.5 Bounded recursion

```text
one high-information question per turn by default
maximum episode depth and age
no repeated question for an already answered field
partial execution for unrelated non-blocked obligations
user may decline; gap remains unresolved without invented meaning
```

## 8. Learning strength, reinforcement, and revision

### 8.1 Evidence weighting

Strong evidence:

```text
explicit definition with grounded target
contrastive example/counterexample
trusted translation alignment
multimodal co-reference with clear event/state
explicit correction of a prior interpretation
independent recurrence across sources/contexts
```

Weak evidence:

```text
single contextual co-occurrence
absence of user correction
repetition from the same source/session
successful surface generation without semantic verification
```

### 8.2 Use-based learning

Every use emits a `LearningUseOutcome`:

```text
binding selected
branch margin
query/write/state result
response success or repair
user confirmation/correction
subsequent reference resolution
```

Successful prediction is support; explicit correction or repair is contradiction. Silence is at most weak support.

### 8.3 Polysemy and context

When contradictory evidence is context-separable, create a new sense rather than weakening both into ambiguity.

Context signatures may include:

```text
language/dialect/domain
neighboring semantic roles
construction ID
entity kinds of arguments
state/relation family
discourse function
modality/polarity/time/place
```

### 8.4 Temporal revision

Word meanings, role holders, preferences, and world facts have different temporal behavior. Lexical stability, world freshness, and user preference recency are separate scores.

### 8.5 Promotion states

```text
observed_candidate
hypothesized
minimally_grounded
session_provisional
user_scoped_active
domain_scoped_active
language_scoped_active
stable
contested
superseded
retired
quarantined
```

Risky operator schemas have a lower promotion ceiling than ordinary lexeme aliases unless independently reviewed.

## 9. Multilingual and multimodal acquisition

### 9.1 Language packs

A language pack supplies only surface mechanics and seed evidence:

```text
tokenization and script normalization
morphological candidates
pronoun/deixis candidates
closed-class candidates
construction candidates
realization grammar
```

It does not own operational routing. Newly learned forms enter the same language-scoped registries.

### 9.2 Cross-language pivot

Different languages share semantic targets:

```text
surface(lang=A) -> sense/operator/state/relation ref <- surface(lang=B)
```

A translation can reinforce a binding while preserving language-specific grammar, sense restrictions, and register.

### 9.3 Multimodal grounding

Visual, audio, sensor, location, temporal, and tool evidence can support the same hypotheses:

```text
spoken/typed word + tracked event -> operator candidate
word + observed object -> entity/concept candidate
modifier + sensor/state change -> state lexeme candidate
preposition + spatial relation -> geospatial relator candidate
```

Every alignment retains modality, timestamp, source, and synchronization confidence.

## 10. Interpretation and learning are branch-coupled

Each interpretation branch contains:

```text
lexical senses
grammatical bindings
construction mapping
entity/referent grounding
predicate activation
scope graph
state/relation consequences
semantic gaps
```

A learning answer updates only branches dependent on the relevant gap. The system does not rebuild unrelated meanings or let one learned word authorize unrelated writes.

Branch scoring combines:

```text
semantic coherence
port/type compatibility
scope consistency
construction support
context/salience
source/evidence quality
learned-binding strength
complexity penalty
```

A weak new binding must not dominate a well-supported established sense without contextual evidence.

## 11. State, causal, and operator learning

### 11.1 State terms

A state word maps to:

```text
holder entity kinds
state family
dimension
value or polarity/intensity
scale/units where relevant
temporal persistence
```

A word such as a newly taught condition cannot be placed in `affective` merely because it describes the user. The learning episode must resolve the actual state coordinate.

### 11.2 Operators

A learned operator schema contains:

```text
operator family
typed ports and optionality
argument realization patterns
preconditions
state deltas
relation deltas
temporal profile
permission/risk metadata
causal/affordance effects
```

Lexical learning may bind a new form to an existing operator with little questioning. Creating a new operator requires more evidence.

### 11.3 Causal learning

Causal hypotheses remain candidates until antecedent/effect, direction, temporal ordering, scope, and confounder policy are adequately supported. They cannot be inferred from one adjacent mention and committed as universal rules.

## 12. Obligations and learning

The obligation graph may contain a `learning_question` node alongside compatible work:

```text
answer known part
+ acknowledge user
+ ask one targeted question for unknown blocking part
```

Clarification preempts only dependent nodes. Safety and permission may preempt more broadly.

```python
@dataclass(frozen=True, slots=True)
class LearningObligation:
    obligation_id: str
    episode_id: str
    gap_ids: tuple[str, ...]
    question_act: str
    expected_answer_schema: Mapping[str, Any]
    resumes_obligation_ids: tuple[str, ...]
```

## 15. Response and realization

Response formation receives:

```text
selected obligation graph
execution ledger
answer/write/state/action outcomes
active learning obligation
semantic question act
style/temperature state
```

It does not inspect the unknown English word to decide what to ask. It realizes a structured question act with slots such as:

```text
unknown_surface
candidate_kind
missing_role
candidate_examples
```

Newly learned realization forms are provisional language resources and cannot reroute semantics.

## 16. Safety, trust, privacy, and poisoning resistance

1. Learning a meaning does not authorize performing it.
2. Safety is evaluated over selected scoped meaning and transmutations, not lexical presence alone.
3. One user cannot silently promote a binding to global scope.
4. Foundational atom kinds, edge types, core permission gates, and safety invariants are not online-learnable.
5. High-risk operator/schema candidates remain quarantined or low-scope until independent validation.
6. Source trust and semantic confidence are separate.
7. Private/user-specific vocabulary remains scoped and permissioned.
8. Store only required surface forms and abstract context/provenance; raw utterance retention follows explicit privacy policy.
9. Coordinated/repeated evidence is dependency-discounted.
10. Learned content never bypasses write, query, tool, or action contracts.

## 17. Efficiency model

### Fast path

```text
compiled language trie
+ session provisional overlay
+ sense/context index
+ construction signature cache
-> bounded candidate generation
```

Known utterances do not enter the learning planner.

### Unknown path

Only blocking or high-value gaps open/continue an episode. Nonessential unknowns remain unresolved while compatible obligations execute.

### Question budget

Select one question by information gain. Do not ask a full dictionary interview when a single alias binding is sufficient.

### Consolidation

Evidence aggregation, schema induction, sense splitting, and global promotion run asynchronously or under explicit budget. The hot path only appends evidence and updates scoped materialized views.

### Complexity targets

```text
lexical lookup: O(tokens * bounded_ngram)
branch generation: bounded beam
question selection: O(active_gaps * candidate_questions)
evidence append: O(1) amortized
strength materialization: incremental
normal durable query: indexed, no broad scan
```

## 19. Non-negotiable invariants

```text
No unknown span is silently assigned operational authority.
No lexical alias directly activates an operator.
No required unresolved port is replaced by an invented entity.
No learning question is chosen from raw-language phrase cases.
No durable learning mutation occurs without LearningContract + GraphPatch validation.
No newly learned meaning grants permission or tool authority.
No single scalar confidence erases source, contradiction, or context history.
No contradiction destructively overwrites prior evidence.
No user-scoped vocabulary silently becomes global.
No word is forced into one sense when context supports polysemy.
No grammar marker is discarded merely because it is closed-class.
No state word is assigned a family/dimension without schema evidence.
No learning patch is emitted by MeaningGraphBuilder.
No response layer decides what an unknown word means.
No successful-use signal is treated as strong truth confirmation by itself.
No safety/provenance/permission gate is removed by budget pressure.
```

## 20. Canonical 3.3 contract

```text
UOL graph is the evidence-preserving semantic workbench.
Interpretation branches are hypotheses, not facts.
Semantic gaps make missing meaning explicit.
Learning episodes recursively acquire only missing fundamental substrate fields.
Provisional bindings enable immediate scoped reuse.
Evidence ledgers govern reinforcement, contradiction, scope, and promotion.
State transmutations are the only state-change authority units.
Obligation graphs preserve compatible meanings and learning needs.
Contracts are the only execution and learning authority.
Graph patches are the only durable semantic mutation path.
Response formation renders selected evidence, outcomes, and semantic question acts.
```
