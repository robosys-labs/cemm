# CEMM 3.3 UOL Graph Architecture

Version: 3.3 revised  
Status: aligned with `cemm/ARCHITECTURE.md` v3.1  
Audience: implementers of `cemm/types/uol_graph.py`, `meaning_graph_builder.py`, concept resolution, port resolution, and training export

## 1. Architectural Correction

The older 3.3 position was close but too strong in one important way:

```text
canonical training and reasoning substrate = UOLGraph
```

The corrected architecture is:

```text
online perception and reasoning substrate = UOLGraph
durable learning mutation format = GraphPatch
durable compressed semantic memory = concept/construction/predicate/affordance lattices
```

`UOLGraph` is the hot-path working graph for one signal, turn, transcript segment, or interpretation window.

It is not the permanent ontology.

It is not the long-term memory format.

It is not where concepts like `person`, `country`, `leader`, `president`, `cold`, `office_role`, or `weather` become native primitives.

Those are learned or seeded concept atoms in the concept lattice.

## 2. Core Contract

The runtime contract is:

```text
raw signal
-> meaning groups
-> candidate UOL atoms and edges
-> concept resolution
-> operational port resolution
-> affordance prediction
-> act planning
-> graph patch extraction
-> asynchronous consolidation
```

The graph is allowed to be uncertain, ambiguous, partial, and multi-hypothesis.

The graph must never pretend that:

```text
one utterance = one intent
one sentence = one predicate
one term = one fixed meaning
one graph = one durable truth
```

## 3. Non-Negotiable Rules

### 3.1 UOL Atom Kinds Are Kernel Categories

The graph uses only these atom kinds:

| Atom Kind | Runtime Meaning |
|---|---|
| `entity` | A referable thing, participant, concept mention, object, organization, person-like thing, or memory target. |
| `process` | Event, process, relation-bearing action, change, doing, becoming, or happening. |
| `state` | Condition, status, feeling, value, disposition, or temporary situation. |
| `relation` | Linkable semantic relation such as `is_a`, `same_as`, `part_of`, or `has_property`. |
| `quality` | Attribute, descriptor, degree-like property, or evaluative feature. |
| `quantity` | Count, degree, amount, measure, threshold, or intensity. |
| `time` | Temporal reference, ordering, freshness marker, duration, recurrence, or scope. |
| `place` | Location, deictic place, spatial region, or world place. |
| `intent` | Communicative or goal-directed orientation. |
| `need` | Missing resource, pressure, requirement, desire, or repair need. |
| `modality` | Possibility, obligation, permission, negation, desire, hypotheticality, or certainty. |
| `evidence` | Evidence requirement, evidence object, verification state, or support marker. |
| `source` | Speaker, memory, document, tool, sensor, self-observation, or world source. |
| `permission` | Scope allowing use, storage, retrieval, action, or disclosure. |
| `action` | Executable or agent-directed action. |
| `self` | CEMM self-reference or self-model target. |

These are representation categories, not an ontology of the world.

### 3.2 Domain Concepts Are Dynamic Keys

`UOLAtom.kind` is fixed.

`UOLAtom.key` is dynamic.

Example:

```text
kind = entity
key = president
```

means:

```text
the graph contains an entity-kind mention whose concept key is president
```

It does not mean:

```text
president is a kernel type
```

The same rule applies to:

```text
person
country
organization
office_role
leader
president
cold
weather
government
authority
```

### 3.3 Edges Are Structural Graph Relations

The graph edge vocabulary should stay small and stable.

Canonical edge types:

| Edge | Use |
|---|---|
| `has_role` | Binds an atom to a participant role, temporary role name, or operational port candidate. |
| `modifies` | Quality, quantity, modality, time, or place modifies another atom. |
| `refers_to` | Surface/source/evidence points at another atom. |
| `asks_about` | Intent asks about an entity, state, process, relation, evidence, time, or place. |
| `teaches` | Source presents a candidate relation, concept, predicate, or rule as learnable. |
| `evaluates` | Valence, judgment, preference, or affect evaluates a target. |
| `causes` | Process/action/state causes a state, relation, process, or need shift. |
| `enables` | Permission, state, process, tool, or condition enables another atom. |
| `prevents` | State, process, permission, lack, or condition prevents another atom. |
| `before` | Temporal ordering. |
| `after` | Temporal ordering. |
| `same_as` | Identity, synonymy, aliasing, or co-reference candidate. |
| `is_a` | Category, inheritance, or class membership candidate. |
| `part_of` | Mereological relation or contained membership. |
| `used_for` | Functional affordance candidate. |
| `has_property` | Entity/state/property relation. |

Do not create a new edge type for every predicate.

Use:

```text
edge_type = has_role
edge.features.role_key = holder
edge.features.port_key = holder
```

or:

```text
edge_type = relation
atom.key = held_by
```

then let predicate schemas and port bindings carry the higher-level operational meaning.

## 4. Required Runtime Objects

### 4.1 UOLGraph

`UOLGraph` is temporary working memory.

Recommended shape:

```python
@dataclass
class UOLGraph:
    id: str
    signal_id: str
    context_id: str
    raw_text: str
    language: str
    atoms: dict[str, UOLAtom]
    edges: list[UOLEdge]
    groups: list[MeaningGroup]
    candidate_sets: list[CandidateSet]
    construction_matches: list[ConstructionMatch]
    concept_resolutions: list[ConceptResolution]
    port_bindings: list[PortBinding]
    affordance_predictions: list[AffordancePrediction]
    evidence: list[EvidenceRef]
    trace: dict[str, Any]
```

The graph expires after the runtime window unless it is retained as a small high-value exemplar.

### 4.2 UOLAtom

```python
@dataclass
class UOLAtom:
    id: str
    kind: AtomKind
    key: str
    surface: str | None = None
    group_id: str | None = None
    span: TextSpan | None = None
    value: Any | None = None
    features: dict[str, Any] = field(default_factory=dict)
    confidence: float = 0.0
    source_refs: list[str] = field(default_factory=list)
    evidence_refs: list[str] = field(default_factory=list)
```

Implementation rule:

```text
kind decides representation category
key points into the dynamic concept lattice when possible
surface preserves the observed form
features are local interpretation data, not durable truth
```

### 4.3 UOLEdge

```python
@dataclass
class UOLEdge:
    id: str
    type: EdgeType
    source_atom_id: str
    target_atom_id: str
    group_id: str | None = None
    features: dict[str, Any] = field(default_factory=dict)
    confidence: float = 0.0
    evidence_refs: list[str] = field(default_factory=list)
```

Edges may carry local `role_key`, `predicate_key`, `scope`, or `negated` features.

Those features do not make new kernel primitives.

### 4.4 MeaningGroup

A meaning group is a probable local semantic unit.

```python
@dataclass
class MeaningGroup:
    id: str
    parent_group_id: str | None
    span: TextSpan
    surface: str
    function:
        Literal[
            "assertion",
            "question",
            "command",
            "teaching",
            "repair",
            "social",
            "evaluation",
            "condition",
            "unknown",
        ]
    atom_ids: list[str]
    edge_ids: list[str]
    confidence: float
```

One utterance may produce many groups.

Example:

```text
I am cold, and can you check the weather?
```

must produce at least:

```text
group_1: assertion/state report: I am cold
group_2: request/fresh-world query: check the weather
```

### 4.5 CandidateSet

The graph must preserve competing interpretations when confidence is not decisive.

```python
@dataclass
class CandidateSet:
    id: str
    target_span: TextSpan
    candidates: list[str]
    selected_atom_id: str | None
    reason: str
```

This prevents the system from collapsing ambiguous utterances too early.

### 4.6 ConceptResolution

Concept resolution connects graph atoms to the durable concept lattice.

```python
@dataclass
class ConceptResolution:
    atom_id: str
    concept_id: str | None
    state:
        Literal[
            "exact_alias",
            "construction_hint",
            "parent_inferred",
            "nearest_fingerprint",
            "new_candidate",
            "unresolved",
        ]
    inherited_from: list[str]
    confidence: float
    evidence_refs: list[str]
```

Unknown terms should create candidate atoms, not errors.

### 4.7 PortBinding

Operational ports are not static slots.

They are resolved dynamically against the graph and concept lattice.

```python
@dataclass
class PortBinding:
    owner_atom_id: str
    owner_concept_id: str | None
    port_id: str | None
    port_key: str
    filler_atom_id: str | None
    required: bool
    status: Literal["bound", "placeholder", "rejected", "ambiguous"]
    score: float
    score_parts: dict[str, float]
    evidence_refs: list[str]
```

The graph can represent an unresolved required port without blocking the whole turn.

The planner asks for repair only when action depends on that missing certainty.

### 4.8 AffordancePrediction

Effects are predictions over bound graph patterns.

They are not ports.

```python
@dataclass
class AffordancePrediction:
    affordance_id: str
    trigger_atom_ids: list[str]
    required_binding_ids: list[str]
    predicted_patch_template: str
    effect_type:
        Literal[
            "state_change",
            "need_activation",
            "action_enablement",
            "action_prevention",
            "evaluation_shift",
        ]
    confidence: float
    evidence_refs: list[str]
```

Bad:

```text
cold.possible_effect = discomfort
```

Good:

```text
cold + holder=user_environment + domain=temperature + intensity=high
-> predicts discomfort_risk
```

## 5. Graph Builder Pipeline

The UOL graph builder should implement this local sequence:

```text
MeaningPerceptPacket
-> normalize spans and groups
-> create source/time/permission/evidence atoms
-> create candidate content atoms
-> create structural edges
-> attach construction matches
-> resolve concepts
-> resolve ports
-> predict affordances
-> attach planner-facing evidence summary
-> export graph patch candidates
```

### 5.1 Source, Time, Evidence, And Permission Atoms

Every graph should include explicit atoms or refs for:

```text
who said it
where it came from
when it was observed
what permission scope applies
what evidence type supports it
whether freshness is required
```

This fixes failures where CEMM cannot distinguish:

```text
the user taught me X
the world currently verifies X
I inferred X
I should not store X
I can use X only in this conversation
```

### 5.2 Content Atoms

Content atoms should be created from:

```text
surface spans
construction hints
known aliases
concept lattice lookup
language adapter features
retrieval/tool hints when allowed
```

Do not block graph creation on perfect concept lookup.

### 5.3 Structural Edges

Structural edges should be generated before final concept resolution.

Example:

```text
a president is a leader of a country
```

should not become:

```text
EntityAtom(leader_of_country)
```

It should become a decomposed candidate graph:

```text
EntityAtom(president)
EntityAtom(leader)
EntityAtom(country)
RelationAtom(is_a)
RelationAtom(of/domain)

president --is_a--> leader
leader --has_role(role_key=domain)--> country
SourceAtom(user) --teaches--> RelationAtom(is_a)
```

The consolidator may later strengthen:

```text
president inherits leader
leader has domain port
domain accepts country-like concepts
```

but the working graph must not bake that in as a single opaque atom.

### 5.4 Concept Resolution

For each atom:

```text
exact alias lookup
construction context lookup
parent search
fingerprint search
source-backed lookup if allowed
candidate creation if unresolved
```

The output is a `ConceptResolution`.

The atom itself remains a runtime atom.

### 5.5 Port Resolution

Port resolution must not be implemented as domain-specific slot filling.

Bad:

```python
if atom.key == "president":
    bind_president_holder_and_country(...)
```

Good:

```python
ports = concept_lattice.get_ports(resolved_concept)
bindings = port_resolver.resolve(graph, owner_atom, ports, context)
graph.port_bindings.extend(bindings)
```

The resolver scores candidates using:

```text
atom kind match
parent concept match
edge pattern match
construction support
source trust
temporal fit
discourse salience
contradiction penalty
freshness penalty
complexity penalty
```

### 5.6 Affordance Prediction

Affordance prediction consumes:

```text
working graph
concept resolutions
port bindings
construction matches
source/evidence policy
```

It produces tentative predictions, not facts.

The planner may use them to:

```text
show empathy
ask a useful question
avoid unsafe claims
choose a tool
activate a need
create graph patch candidates
```

## 6. GraphPatch Boundary

The only legal path from `UOLGraph` into durable learning is `GraphPatch`.

```python
@dataclass
class GraphPatch:
    id: str
    source_graph_id: str
    target:
        Literal[
            "concept_lattice",
            "construction_lattice",
            "predicate_schema",
            "causal_affordance",
            "episodic_trace",
            "source_policy",
            "discard",
        ]
    operations: list[PatchOperation]
    source_refs: list[str]
    evidence_refs: list[str]
    permission_refs: list[str]
    confidence: float
    reason: str
```

Never write directly from graph interpretation into durable memory.

Bad:

```python
memory.store_fact("Donald Trump is current president of USA")
```

Good:

```python
patch = graph_patch_extractor.extract(graph)
consolidator.score_and_promote(patch)
```

## 7. Planner Contract

`ActResolutionPlanner` should prefer graph evidence over older high-level labels when available.

It should consume:

```text
meaning groups
intent atoms
source/evidence atoms
concept resolutions
port bindings
affordance predictions
freshness requirements
permission scope
conversation context
```

It should output:

```text
reply act
retrieval/tool requirement
memory patch candidates
repair question if needed
trace summary
```

The planner must distinguish:

| Case | Correct Behavior |
|---|---|
| User says "you?" after "how are you?" | Social/self-directed turn, not verified-fact failure. |
| User teaches a definition | Store as sourced candidate graph patch. |
| User asserts a current-world office holder | Store assertion but require fresh evidence before verified claim. |
| User says "it is cold where I am" | Treat as user state/environment report; do not require weather lookup unless asked. |
| User asks live weather | Tool or fresh external source required. |
| User insults CEMM jokingly | Social/evaluative act; respond socially, do not enter evidence abstention loop. |

## 8. Examples

### 8.1 Multi-Group Utterance

Input:

```text
I am cold, and can you check the weather?
```

Expected groups:

```text
group_1:
  function = assertion
  atoms = self/user, cold, present_time
  likely bindings = cold.holder -> user_or_user_environment

group_2:
  function = question/command
  atoms = check, weather, external_world_evidence
  asks_about = weather
  evidence_policy = fresh_source_required
```

Planner:

```text
acknowledge cold as user report
request/use location if weather lookup is available and permitted
do not answer "I don't have enough verified information" to the user report
```

### 8.2 Teaching Definition

Input:

```text
a president is a leader of a country
```

Expected graph:

```text
Entity(president)
Entity(leader)
Entity(country)
Relation(is_a)
Relation(domain/of)
Source(user)
Permission(conversation)

president --is_a--> leader
leader --has_role(role_key=domain)--> country
user --teaches--> is_a
conversation_permission --enables--> use_of_user_assertion
```

Graph patch candidates:

```text
concept: president parent leader
concept: leader port domain accepts country-like concept
predicate: leads(actor, domain) if supported by surrounding evidence
source support: user taught this in conversation
```

### 8.3 Current Holder Claim

Input:

```text
Donald Trump is the current president of the USA
```

Expected graph:

```text
Entity(Donald Trump)
Entity(president)
Entity(USA)
Time(current)
Relation(held_by)
Source(user)

president --has_role(role_key=holder)--> Donald Trump
president --has_role(role_key=domain)--> USA
current --modifies--> held_by
user --teaches/asserts--> held_by
```

Planner:

```text
may acknowledge the user assertion
may store sourced candidate claim
must not present as verified current fact without fresh evidence
```

No hardcoded `president` branch is required.

### 8.4 Cold Report

Input:

```text
coz it's very cold where I am right now
```

Expected graph:

```text
State(cold)
Quantity(very)
Place(where_I_am)
Time(now)
Source(user)

very --modifies--> cold
where_I_am --modifies--> cold
now --modifies--> cold
```

Possible port bindings:

```text
cold.holder -> user_environment
cold.intensity -> very
cold.place -> where_I_am
cold.time -> now
```

Affordance prediction:

```text
temperature_cold + holder=user_environment + intensity=high
-> possible discomfort_risk
```

Planner:

```text
respond conversationally or empathetically
do not require live verification unless the user asks for weather/current external facts
```

## 9. Training Export

The training target is no longer:

```text
surface tokens -> one intent label
```

The target is:

```text
surface tokens + context
-> meaning groups
-> candidate UOL graph
-> concept resolutions
-> port bindings
-> affordance predictions
-> graph patch candidates
```

Recommended supervised or self-supervised targets:

```text
meaning group segmentation
atom kind prediction
atom key normalization
candidate concept linking
edge type prediction
edge endpoint prediction
role/port candidate prediction
port binding score calibration
construction match prediction
freshness/evidence policy prediction
affordance trigger prediction
graph patch extraction
planner consistency
```

Do not train the model to memorize every transcript graph.

Train it to extract reusable structure that consolidates into:

```text
concept atoms
aliases
ports
predicate schemas
causal affordances
construction operators
source/evidence policies
small exemplars
```

Recommended split:

```text
cemm/types/uol_graph.py
  UOLAtom, UOLEdge, UOLGraph, MeaningGroup, CandidateSet

cemm/types/graph_patch.py
  GraphPatch, PatchOperation

cemm/kernel/meaning_perceptor.py
  surface signal -> meaning groups and candidate semantic spans

cemm/kernel/meaning_graph_builder.py
  meaning packet -> temporary UOLGraph

cemm/kernel/construction_matcher.py
  surface/graph signatures -> construction candidates

cemm/kernel/concept_resolver.py
  runtime atoms -> concept lattice candidates

cemm/kernel/port_resolver.py
  concept ports + working graph -> PortBinding records

cemm/kernel/affordance_predictor.py
  bound graph patterns -> tentative effect predictions

cemm/kernel/act_resolution_planner.py
  graph state -> response/tool/memory plan

cemm/learning/graph_patch_extractor.py
  UOLGraph -> graph patch candidates

cemm/learning/concept_consolidator.py
  scored patches -> durable concept lattice
```

## 11. Performance Requirements

The graph is hot-path.

Keep it small, local, and disposable.

Required indexes:

```text
atom_id -> atom
group_id -> atom ids
source_atom_id -> outgoing edges
target_atom_id -> incoming edges
surface span -> atom candidates
concept_id -> resolved atoms
port_key -> candidate bindings
```

Recommended caps:

```text
short chat turn: 10-80 atoms
long user message: segment into groups before graph expansion
transcript study: process in windows and consolidate asynchronously
candidate alternatives: prune low-score branches early
```

Do not preserve every graph.

Preserve:

```text
graph patches
operator traces
counterexamples
small exemplars
training exports
```

## 12. Acceptance Tests

### 12.1 No Domain Primitive Test

Given:

```text
president
country
person
cold
weather
```

Expected:

```text
all appear as UOLAtom.key values or concept lattice keys
none appear as AtomKind values
```

### 12.2 No Static Slot Test

Given:

```text
Donald Trump is the current president of the USA
```

Expected:

```text
holder/domain/time bindings are PortBinding records
bindings are produced by port_resolver
no code path checks if atom.key == "president" to fill slots
```

### 12.3 Multi-Meaning Test

Given:

```text
I am cold, and can you check the weather?
```

Expected:

```text
at least two MeaningGroup records
first group is a user/environment state report
second group is a fresh-world query/action request
planner does not collapse both into one abstention
```

### 12.4 Teaching Test

Given:

```text
a president is a leader of a country
```

Expected:

```text
president, leader, and country are separate atoms
graph patch candidates are produced
durable memory is not directly mutated by graph builder
```

### 12.5 Effect As Affordance Test

Given:

```text
it is very cold where I am
```

Expected:

```text
cold has resolved holder/intensity/place/time bindings where possible
discomfort is an affordance prediction, not cold.possible_effect
```

## 13. Final Implementation Rule

`UOLGraph` is CEMM's temporary semantic workbench.

`GraphPatch` is CEMM's durable learning gate.

The concept lattice, construction lattice, predicate schemas, and causal affordances are CEMM's compressed long-term semantic machinery.

Any implementation that stores UOL graphs as the main memory, hardcodes domain concepts as primitives, fills slots with concept-specific branches, or treats predicted effects as static fields has drifted from the architecture.
