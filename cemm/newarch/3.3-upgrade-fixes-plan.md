# CEMM 3.3 Upgrade: Fixes Plan

**Status:** Current-code remediation plan — learning-integrated revision 2  
**Baseline reviewed:** `robosys-labs/cemm@c6675ee5e822c4af6253e1bce2e76975fed0f9c9`  
**Purpose:** Fix the earliest substrate defects that prevent safe recursive learning, immediate reuse, strength revision, and correct semantic execution.

---

## 1. Priority and repair law

```text
P0 — creates unsupported authority, false learning/write/state, or unsafe interpretation
P1 — blocks recursive acquisition, reuse, revision, or compositionality
P2 — performance, migration, observability, and cleanup
```

Repair the earliest wrong artifact. Do not add transcript phrases to operational compilers, query execution, patch extraction, safety, or response formation.

---

## 2. P0 — Candidate and authority defects

### F-001 — Action aliases directly activate operators

**Current:** `SchemaBackedLanguageAdapter.map_actions()` emits `ActionAtom`; noun-like aliases can activate restricted actions.  
**Fix:** emit `LexicalSemanticCandidate`; activate only after predicate-head, scope, typed ports, polarity, modality, quotation, and illocution resolve.  
**Learning impact:** a newly taught verb first binds to an operator candidate; it does not execute or project state deltas.

### F-002 — Missing actor defaults to user

**Current:** schema role inference can assign the user without evidence.  
**Fix:** unresolved `RoleRef`; only construction-licensed null-subject inference may choose speaker/listener, with provenance and penalty.  
**Learning impact:** recursive question asks for actor only when it blocks grounding.

### F-003 — Role placeholders become object entities

**Current:** unresolved ports can materialize as ordinary entities and appear valid.  
**Fix:** placeholders never satisfy typed ports or state targets. Preserve `PortBinding(status="placeholder")`.  
**Learning impact:** missing role becomes `SemanticGap(gap_kind="required_port")`.

### F-004 — Schema deltas compile before applicability

**Current:** recognized actions immediately create state atoms/deltas.  
**Fix:** compile deltas only from `PredicateActivationFrame` with valid scope/status.  
**Learning impact:** examples of a new operator can describe desired, questioned, hypothetical, reported, or completed transitions distinctly.

### F-005 — Authority-creating fallback frames

**Current:** unmatched meaning may default to social/world-fact/concept-definition lanes.  
**Fix:** emit `UnresolvedOperationalMeaning` + `SemanticGap`; clarify or abstain.  
**Learning impact:** unknown words open acquisition rather than receiving fabricated semantics.

### F-006 — Graph builder emits persistence candidates too early

**Current:** `MeaningGraphBuilder` creates graph patch candidates from surface teaching, state deltas, emotional evaluations, concepts, and remember commands.  
**Fix:** graph builder emits structural propositions and `LearningObservation` only. `LearningPatchCompiler` runs after `LearningContract`.  
**Learning impact:** teaching evidence cannot bypass branch selection, scope, permission, or provenance.

### F-007 — Patch authorization is target-level

**Current:** a writable contract can authorize unrelated same-target operations from suppressed groups.  
**Fix:** each operation carries source frame/group/branch/episode/gap IDs; authorization requires exact authorized frame provenance.  
**Learning impact:** one valid teaching clause cannot authorize a different criticism/social clause.

### F-008 — Surface sanitizers repair semantic segmentation

**Current:** insult-tail/object-surface cleanup in patch extraction rewrites text after interpretation.  
**Fix:** split factual, vocative, evaluation, and discourse groups upstream; delete sanitizer.  
**Learning impact:** store the grounded proposition and separately process evaluation evidence.

### F-009 — State transmutations are not authoritative applied state

**Current:** transmutations can be compiled/used for current-turn routing but not consistently authorized, applied, and persisted.  
**Fix:** policy resolver + authorizer + applier + result ledger; persist session occupancy.  
**Learning impact:** new state words can be tested against subsequent state references and reinforced/corrected.

### F-010 — Contract authority is duplicated

**Current:** obligation builder and specialized builders may create competing subcontracts.  
**Fix:** one `OperationalContractCompiler`, including `LearningContract`.  
**Learning impact:** no parallel path can activate or commit learned artifacts.

## 3. P0 — Learning integrity defects

### F-011 — Unknown tokens are records, not operational gaps

**Current:** perception can preserve unknown tokens, but there is no authoritative gap lifecycle connected to obligations and learning.  
**Fix:** add `SemanticGapDetector`, blocking analysis, episode ownership, and question-plan linkage.  
**Acceptance:** unknown content essential to a query/action creates a focused learning obligation; nonessential unknown content does not block unrelated work.

### F-012 — No first-class recursive `LearningEpisode`

**Current:** teaching/topic state exists, but it does not model arbitrary missing semantic fields, asked questions, evidence, hypotheses, depth, or resume dependencies.  
**Fix:** implement `LearningEpisodeManager` persisted in `SessionStore`.  
**Acceptance:** multi-turn acquisition resumes after each answer and never repeats a satisfied question.

### F-013 — No learning contract or promotion ceiling

**Current:** write contracts cover facts/relations but not scoped vocabulary/schema acquisition.  
**Fix:** `LearningContract` defines permitted operations, provisional scope, evidence requirements, and maximum promotion.  
**Acceptance:** one user's teaching may activate a session/user binding but cannot install a global/high-risk operator.

### F-014 — Confidence lacks an append-only strength trace

**Current risk:** direct confidence updates can lose which source, context, contradiction, and use outcome produced the number.  
**Fix:** append `LearningEvidenceEvent`; derive `KnowledgeStrength` materialized view.  
**Acceptance:** confidence is reproducible; duplicate same-source events are discounted; contradiction remains visible.

### F-015 — Learning meaning and accepting proposition truth can collapse

**Fix:** separate lexical/schema binding evidence from world-claim evidence and storage.  
**Acceptance:** teaching a word's meaning does not validate all claims expressed with that word.

### F-016 — Provisional learning is not guaranteed immediately reusable

**Fix:** add session learning overlay consulted before durable registries; activate after minimum grounding via contract; persist across turns.  
**Acceptance:** newly taught form is understood on the next turn without waiting for global consolidation.

### F-017 — Contradiction may overwrite instead of branch/revise

**Fix:** support reinforce, contradict, restrict, split sense, supersede, retire, quarantine.  
**Acceptance:** polysemy creates context-conditioned senses; temporal change supersedes with history.

### F-018 — Successful use may be overcounted

**Fix:** `LearningUseOutcome`; explicit confirmation/correction strong, silence weak, same-session recurrence dependent.  
**Acceptance:** repeated use by one source cannot rapidly promote global confidence.

### F-019 — Foundational schemas are too easy to extend

**Fix:** online learning can bind new forms to existing substrates freely within scope; new state dimensions/entity kinds/high-risk operators require schema-candidate quarantine and stricter promotion.  
**Acceptance:** user teaching cannot add UOL primitive kinds, edge types, permissions, tools, or safety exceptions.

## 4. P1 — Multilingual lexical and grammar learning

### F-020 — Hardcoded ambiguity table in runtime code

**Current:** `MeaningPerceptor._AMBIGUOUS_LEXEMES` contains English examples.  
**Fix:** move to versioned language data or learned sense registry; all ambiguity remains candidate-level.  
**Acceptance:** same mechanism supports any language/dialect.

### F-021 — Surface teaching parser is English-specific and structurally authoritative

**Current:** `_add_surface_teaching_relations`, possessive maps, contractions, stop-word cleaning, and cue parsing live in `MeaningGraphBuilder`.  
**Fix:** language construction schemas produce relation candidates; graph builder only compiles supplied structures.  
**Acceptance:** multilingual possessives/copulas/case markers can express profile and concept relations without core-code word tables.

### F-022 — Remember-command extraction reparses raw English

**Current:** `_extract_remember_relation_patches()` tokenizes and searches for `remember`.  
**Fix:** memory command becomes an operator/obligation over an embedded proposition branch.  
**Acceptance:** every supported language uses its own lexical/grammar binding while core memory write logic remains language-neutral.

### F-023 — Implicit predicate detector inspects English pronouns

**Current:** token sets such as `i/we/my` and `you/your` determine actor.  
**Fix:** consume resolved referent/grammatical-role candidates from the language layer.  
**Acceptance:** pro-drop, inflected, and non-English languages use the same semantic resolver.

### F-024 — Articles and prepositions are treated as stop words or cue fragments

**Fix:** add `GrammarOperatorBinding` and grammatical-function gaps.  
**Acceptance:** determiners affect specificity/quantity; relators affect temporal/geospatial/instrument/source/goal/domain roles.

### F-025 — Construction learning is raw-pattern memorization risk

**Fix:** induce construction schemas over semantic slot categories, ordering/dependency constraints, and UOL graph rewrites. Require lexical diversity before generalization.  
**Acceptance:** learned construction applies to unseen fillers but does not overmatch unrelated clauses.

### F-026 — Language identity and dialect/domain scope are incomplete

**Fix:** every binding has language/script/dialect/domain/register and user scope.  
**Acceptance:** same surface in two languages or dialects may map differently without collision.

## 5. P1 — Semantic schema acquisition

### F-027 — Predicate outcomes are hardcoded by action key

**Current:** `PredicatePhraseExtractor._outcome_for_predicate()` switches on selected action/intent keys.  
**Fix:** derive expected outcomes from operator/state/relation schemas and scoped activation.  
**Learning impact:** learned operators automatically participate after schema validation; no new code branch per action.

### F-028 — New operator learning lacks typed minimum criteria

**Fix:** require operator family, typed ports, at least one effect/relation delta, modality behavior, and permission/risk policy before provisional activation.  
**Acceptance:** alias-to-known-operator is cheap; new operator creation asks recursively for missing fields.

### F-029 — State terms can be forced into the wrong family

**Fix:** `StateCoordinateResolver` plus state-learning episode for holder/family/dimension/value/polarity/scale.  
**Acceptance:** physical, vital, affective, cognitive, capability, geospatial, temporal, social, permission, informational, and operational states remain distinct.

### F-030 — Relation/preposition orientation is under-specified

**Fix:** learn source/target roles, relation family, inverse, symmetry, transitivity/composition policy, and context restrictions.  
**Acceptance:** a learned relator cannot silently reverse source and target.

### F-031 — Causal learning lacks applicability and evidence diversity

**Fix:** causal-rule candidates require antecedent/effect, temporal order, scope, source independence, and conflict trace.  
**Acceptance:** one co-occurrence does not become a universal causal rule.

### F-032 — Cross-language learning may conflate translation with identity

**Fix:** translation is evidence linking language-specific senses to a shared target, not unconditional `same_as`.  
**Acceptance:** partial translations and language-specific senses remain representable.

### F-033 — Multimodal evidence is not unified with learning hypotheses

**Fix:** modality-neutral evidence events with synchronization confidence and shared typed refs.  
**Acceptance:** visual/audio/sensor examples reinforce the same hypothesis ledger without special durable paths.

## 6. P1 — Obligation, response, and session defects

### F-034 — Single-primary arbitration drops compatible meanings

**Fix:** obligation graph with dependencies and co-primary nodes.  
**Learning impact:** answer known content while asking one targeted question for the unknown dependent part.

### F-035 — Semantic program ranking uses surface token length

**Current:** information content is approximated by surface word count.  
**Fix:** rank from authority, obligation force, branch coverage, novelty, unresolved blocking load, and evidence—not language-dependent length.

### F-036 — Clarification is generic rather than field-specific

**Fix:** semantic question acts selected by expected information gain; NLG supplies wording.  
**Acceptance:** ask for the exact missing kind/port/effect/relation/state/time/place/grammar field.

### F-037 — Output state reparses generated English

**Current:** `OutputStateUpdater` regexes output text to infer question type.  
**Fix:** realization emits `OutputActFrame` and expected-answer schema.  
**Learning impact:** active episode resumes from structured output metadata in any language.

### F-038 — Session store omits full learning/state continuation

**Fix:** persist active episodes, gaps, provisional overlays, asked fields, expected answers, entity occupancy, style/repair state, topic/referent salience, and expiration.  
**Acceptance:** restart or next turn resumes the exact acquisition state.

### F-039 — Response fallback strings bypass language realization

**Fix:** blocked/abstain/clarify/learning-question are normal semantic moves.  
**Acceptance:** all supported languages realize them through language packs.

## 7. P2 — Storage, performance, and consolidation

### F-040 — No dedicated hot indexes for learned language

Add:

```text
(language, normalized_form)
(language, lemma, grammatical_category)
(context_signature, sense_id)
(semantic_target, language)
(construction_signature, language)
```

### F-041 — Strength recomputation may become expensive

Use append-only events plus incremental materialized projections; periodic full verification in background.

### F-042 — Evidence dependence not indexed

Index `independence_key`, source, session, context cluster, and time bucket to discount copied/repeated evidence.

### F-043 — Consolidation can interfere with hot path

Move sense splitting, schema induction, global promotion, and historical compaction to bounded background work. Hot path appends evidence and updates session/user projections.

### F-044 — Raw context retention risks privacy and storage growth

Store required surface forms, span refs, abstract context signatures, and policy-approved exemplars. Raw utterance retention requires explicit episodic/privacy policy.

### F-045 — No learning-specific performance gates

Track:

```text
known-input learning overhead
unknown gap detection latency
question-plan latency
overlay lookup latency
evidence append latency
branch count and episode depth
memory per active episode
background consolidation queue depth
```

## 9. Required regression traces

```text
new noun alias -> one-question grounding -> immediate reuse
new verb alias to known operator -> reuse with correct ports
new operator -> recursive actor/target/effect acquisition
new state word -> correct state coordinate and holder
new preposition -> relation orientation from contrastive examples
new article/determiner -> grammatical feature, not domain concept
polysemous word -> two context-conditioned senses
explicit correction -> contradiction event and revision
same-source repetition -> discounted support
cross-language translation -> shared target, separate grammar
multimodal example -> evidence event with synchronization trace
unknown non-blocking word -> partial answer still executes
user declines teaching -> no fabricated meaning or write
learned risky term -> understood, action still safety/permission gated
session slang -> no global promotion
restart after durable promotion -> learned meaning remains usable
```

## 10. Fix completion gate

```text
Unknown language is represented, localized, and questioned—not guessed.
Learning recursively reaches entity/relation/operator/state/time/place/grammar substrates.
Minimum grounding enables immediate scoped reuse.
Every learned artifact retains source, strength, context, and contradiction history.
Future conversation updates strength through explicit use outcomes.
Polysemy and temporal change are non-destructive.
No learned binding bypasses safety, permission, write, query, or action authority.
No core runtime component contains transcript-specific teaching or clarification phrases.
```
