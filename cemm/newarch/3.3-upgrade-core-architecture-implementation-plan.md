# CEMM 3.3 Upgrade: Core Architecture Implementation Plan

**Status:** Proposed implementation contract — learning-integrated revision 2  
**Baseline reviewed:** `robosys-labs/cemm@c6675ee5e822c4af6253e1bce2e76975fed0f9c9`  
**Target:** Deliver the authority-preserving execution spine and recursive multilingual learning spine together, without a late bolt-on learning subsystem.

---

## 1. Delivery principles

```text
observability first
-> candidate/authority separation
-> semantic gaps and learning episodes
-> scoped grounding and operator activation
-> state/causal/obligation authority
-> single execution + learning contracts
-> provisional reuse
-> durable revision/consolidation
-> legacy deletion
```

Rules:

1. Every phase delivers both runtime interpretation and learning consequences where relevant.
2. Remove a fallback only after unresolved meaning is representable.
3. Do not enable durable 3.3 learning before provenance-bound patch authorization exists.
4. Provisional session learning may ship earlier because it is reversible and scoped.
5. Golden tests assert internal traces before transcript wording.
6. Known-input latency must not regress materially.

---

## 2. Parallel workstreams

| Workstream | Responsibility |
|---|---|
| A — Semantic authority | evidence, branches, grounding, scoped operator/state activation |
| B — Recursive acquisition | gaps, episodes, question planning, answer assimilation, provisional reuse |
| C — State and causality | occupancy, deltas, transmutations, effects, safety |
| D — Contracts and execution | obligation graph, query/write/reaction/state/learning contracts, ledger |
| E — Durable learning | evidence ledger, revision, promotion, consolidation, indexes |
| F — Language and NLG | multilingual grammar candidates, semantic clarification acts, realization |
| G — Conformance/performance | traces, fuzzing, poisoning tests, latency and memory budgets |

No workstream may bypass another's authority boundary.

---

## 3. Rollout model

Feature flags:

```python
semantic_spine_version = "3.2"          # 3.2 | 3.3-shadow | 3.3
learning_spine_version = "legacy"       # legacy | 3.3-shadow | 3.3
provisional_learning_enabled = False
learning_commit_authority = "disabled"  # disabled | shadow | 3.3
```

Stages:

```text
1. 3.3 trace-only; no behavior or state change.
2. Candidate/gap shadow; compare with legacy unknown handling.
3. Session LearningEpisode and provisional overlay authority.
4. 3.3 interpretation/query authority; durable learning still shadow.
5. 3.3 state/obligation/contract authority.
6. Durable learning patch authority; never dual-commit.
7. 3.3 response authority.
8. Delete legacy surface parsers and direct learning writes.
```

## 21. Required acceptance suites

### Vocabulary acquisition

```text
teach alias to known concept -> immediate session reuse
restart -> durable reuse after promotion
new word used in different context -> correct sense selection
explicit correction -> old sense weakened or split, not erased
same-source repetition -> limited strength increase
```

### Recursive operator learning

```text
unknown predicate -> ask kind/effect/ports as needed
answer actor but not target -> ask only target/effect still missing
minimum grounding -> provisional use
unsafe meaning learned -> understood but execution remains gated
```

### Grammar learning

```text
unknown preposition -> relation-orientation episode
contrastive examples -> scoped relation binding
article/determiner -> definiteness/quantity feature, not concept atom
learned construction -> generalizes to new lexical fillers
```

### Multilingual/multimodal

```text
translation creates shared semantic target with separate surface bindings
same form in two languages remains distinct
visual example grounds entity/state/operator candidate
source/time/modality trace survives
```

### Revision/security

```text
user-scoped slang never becomes global automatically
conflicting trusted sources create contested/superseded state
poisoning repetition is dependency-discounted
learning cannot grant tool permission
private vocabulary obeys scope and retention policy
```

### Performance

```text
known input does not invoke learning question planner
unknown non-blocking token does not stall unrelated obligations
branch/question budgets are enforced
no broad durable scans
strength updates are incremental
```

## 22. Definition of done

```text
A supported language can introduce an unseen surface form.
CEMM creates a typed gap rather than guessing authority.
CEMM asks the minimum useful semantic question recursively.
The answer fills entity/relation/operator/state/time/place/grammar fields.
A minimally grounded binding becomes immediately usable within scope.
Every later use produces traceable support, contradiction, or restriction evidence.
Knowledge strength and scope are derived from an append-only ledger.
Polysemy, correction, temporal change, and source conflict are non-destructive.
Durable promotion occurs only through authorized graph patches.
NLG remains downstream and language-specific; semantic authority remains language-neutral.
```
